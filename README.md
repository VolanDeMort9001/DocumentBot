## Возможности пользователя.

Бот создает отчеты под шаблон. У бота есть заданный набор шаблонов, которые можно вставить в документ, а также общий шаблон, создающий общее оформление текста.

## Старт бота

1. Пользователь вводит команду /start для запуска бота. После этого бот будет работать вечно до перезапуска кода.

## Функционал бота

1. Всегда пока бот запущен пользователь видит сообщение для выбора шаблона, который можно добавить в документ, через inline-кнопки.
2. Если выбран хотя бы один шаблон, есть кнопка "Завершить отчет". При ее нажатии пользователь получает готовый отчет, а все выбранные шаблоны сбрасываются, пользователь сразу получает сообщение для выбора новых шаблонов для нового отчета.
3. Когда пользователь выбирает один шаблон, он получает новое сообщение с обновленными кнопками, но без уже выбранных шаблонов и с кнопкой "Завершить отчет". Так не будет повторения шаблонов.

## models

* models.py - файл, в котором содержится код формирования файла документа с помощью python-docx.

## commands 

* document.py - единственная команда, создание текста документа из шаблонов.

## infra

## main.py

* Точка входа в бот.

## app.py

* Создает бота, диспетчер и хендлеры. Запускает работу бота

## routers/router.py

* Код рутеров бота. Обменивается с пользователем сообщениями и через FSMContext хранит и передает оставшиеся невыбранные шаблоны.

## keyboards/keyboard.py

* Код клавиатуры бота, который по списку из номеров оставшихся шаблонов создает их клавиатуру и по необходимости добавляет кнопку "завершить отчет"
